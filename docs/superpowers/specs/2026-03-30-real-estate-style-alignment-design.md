# 不动产模块前端样式统一对标设计规范

## 1. 背景与目标
为了确保资产管理系统中不同模块（固定资产与不动产）的品牌感与操作体验一致，特制定本规范。所有不动产相关页面必须 100% 对照 `/asset/info` 系列页面的视觉语言进行重构。

## 2. 列表页规范 (ListView)
### 2.1 布局架构
- **容器**：使用 `.real-estate-page.art-full-height` 搭配 `flex` 布局。
- **侧边栏**：引入 `InfoFilterTree` 组件，默认展示不动产位置树，宽度固定为 240px。
- **主区域**：右侧主体使用 `.real-estate-main`，包含 `ArtSearchBar` 和封装在 `ElCard(shadow="never")` 中的 `ArtTable`。

### 2.2 组件对标
- **ArtSearchBar**：搜索项标签必须精简，且操作按钮（搜索/重置）位于右侧对齐。
- **ArtTableHeader**：操作按钮顺序统一为：[新增, 导入, 导出, 删除]。按钮必须带有 `v-ripple`。

## 3. 详情/编辑页规范 (DetailView)
### 3.1 Hero Header 规范
- **Eyebrow**：展示子模块名（如：“不动产登记”）。
- **Title**：展示页面动作（如：“新增不动产”、“编辑不动产”）。
- **Description**：统一使用引导性文案：“先把不动产主档录稳，后续单据、事件和附件都沿用这套基础字段继续流转。”

### 3.2 表单卡片 (Section Cards)
- **头部结构**：每个 `ElCard` 的 `header` 插槽必须包含三层文字：
  1. `.eyebrow` (灰色小字，如：登记)
  2. `.title` (加黑标题，如：展示基础信息)
  3. `.desc` (淡灰色描述，如：描述不动产的核心地理及分类属性)
- **栅格规范**：表单字段统一使用 `ElRow` + `ElCol`，在 PC 端以 12 栅格或 24 栅格布局，保持纵向对齐。

### 3.3 底部操作栏
- 使用绝对定位固定在底部。
- 按钮对齐方式与固定资产模块保持 1:1 同步：[返回 (plain), 保存 (primary), 保存并继续 (success)]。

## 4. 颜色与投影
- **背景**：统一使用 `var(--art-bg-color)`。
- **卡片**：所有卡片统一设置 `shadow="never"`，边框颜色遵循系统 Token。
- **高亮**：不动产相关图标和强调字统一使用系统主色调。
