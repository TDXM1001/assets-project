# Sprint Plan: 资产管理系统 Sprint 3 - 页面化与模板化

Generated by /sprint-planning on 2026-03-26
Branch: main
Repo: assets-project
Status: DRAFT

## Sprint Goal

把复杂新增流从弹窗升级为独立页面，同时建立分类字段模板底座，让资产系统从“固定资产 MVP”继续演进为“可扩展资产台账平台”。

## Planning Summary

- 项目当前已经完成分类、位置、台账、单据、盘点、维修、事件流水、看板和附件的主链路。
- 下一轮不再以零碎 UI 修补为主，而是优先补齐页面化和模板化能力。
- 本 Sprint 的目标是把新增资产、单据新增和分类字段模板做成可交付、可复用的系统能力。

## Story List

| Story ID | Story | Priority | Points | Notes |
|---|---|---:|---:|---|
| STORY-E006-S001 | 资产新增页面化 | Must Have | 8 | 从弹窗升级为独立页面/向导页 |
| STORY-E006-S002 | 单据新增页面化 | Must Have | 8 | 入库、领用、调拨、归还、报废统一入口 |
| STORY-E006-S003 | 分类字段模板落地 | Must Have | 5 | 分类差异字段受控配置与渲染 |

## Recommended Execution Order

1. `STORY-E006-S003` 先把分类字段模板和渲染底座跑起来。
2. `STORY-E006-S001` 再把资产新增页切到独立页面，并接入模板扩展区。
3. `STORY-E006-S002` 最后把单据新增页页面化，统一头信息、明细和草稿/提交流程。

## Existing Foundations

- `asset_category` 已经承载分类规则。
- `asset_info` 已经具备台账核心字段和 CRUD 链路。
- `asset_order` 已经具备单据头 + 明细的数据模型。
- `asset_attachment` 已经具备附件入口能力。
- `asset/event` 和 `asset/dashboard` 已经证明系统主链路和可观测能力可用。

## Out of Scope This Sprint

- 多资产维修
- 盘点增强
- 顶部 / 行内操作栏统一规范
- 资产类型扩展到不动产 / 低值易耗品

## Done Criteria

- 资产新增和单据新增都能从独立页面完成核心流程。
- 分类字段模板可以配置字段编码、组件类型、必填、字典来源、默认值和版本。
- 新页面能够在不破坏现有列表、详情、导出和权限口径的前提下上线。
- 后续的多资产维修和资产类型扩展能够复用这轮建立的页面化与模板化底座。
