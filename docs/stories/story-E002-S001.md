# 入库与领用单据闭环

**ID：** STORY-E002-S001
**史诗：** E002 单据主链路
**优先级：** Must Have
**故事点：** 5

## 用户故事

作为资产管理员，我希望完成入库与领用单据闭环，以便资产能够从建账走到审批、发放和台账同步更新。

## 验收标准

- [x] 可以创建、保存草稿、提交、审批、驳回、完成和取消单据。
- [x] 入库完成后会生成资产主档并写入事件流水。
- [x] 领用完成后责任人、部门、位置和状态会同步更新。
- [x] 单据流转过程中的状态标签和动作按钮正确显示。

## 技术说明

- 沿用 `asset/order` 模块完成通用单据骨架。
- 需要确保单据完成动作与资产主档回写在同一事务内。
- 事件流水必须记录关键前后值，便于追溯。

## 任务分解

- [x] 完成后端单据 CRUD 与提交/审批/完成/取消接口。
- [x] 完成入库与领用的状态机和资产回写逻辑。
- [x] 完成前端单据列表、新建弹窗和审批弹窗。
- [x] 完成事件流水联动和联调验证。

## 依赖

- 资产台账先可用。

## 实现记录

- 当前方案沿用现有资产主档与事件流水回写链路完成入库与领用闭环，不额外拆出一套独立的“入库造档”分支。
- 后端单据状态流转、资产快照回写、事件流水记录、前端状态按钮展示都已落地。
- 已核对并执行与本 story 相关的增量 SQL，当前本地数据库口径已一致。
