# 上下文回流与草稿隔离
**ID:** STORY-E008-S002  
**史诗:** E008 单据工作台与流程连续性  
**优先级:** Must Have  
**故事点:** 5

## 用户故事

作为资产管理员，我希望从列表、维修、报废等入口进入单据创建页时能够保留来源上下文和草稿隔离，以便返回时不丢失筛选状态，也不会串草稿。

## 验收标准

- [ ] 列表页进入创建页时，能够携带 `orderType`、`bridgeSource`、`bridgeKey`、`sourceBizType`、`sourceBizId` 和 `repairId` 等上下文。
- [ ] 创建页本地草稿按上下文隔离，不同来源不会互相覆盖。
- [ ] 保存成功后返回列表时，能够恢复原来的单据类型和筛选条件。
- [ ] 直接访问创建页时，页面能够自动降级到通用新建单据，不会报错。
- [ ] 来源业务失效或上下文缺失时，页面能够给出明确回退路径。

## 技术说明

- 建议抽出统一的上下文解析与草稿键生成逻辑，避免散落在多个页面里。
- 草稿键不能只按单据类型保存，否则维修、报废和通用单据会互相污染。
- 返回列表时应该优先恢复来源筛选态，再进入列表视图。
