# 资产模块操作栏统一

**ID:** STORY-E007-S001  
**史诗:** E007 交互一致性与操作规范  
**优先级:** Must Have  
**故事点:** 5

## 用户故事

作为资产管理人员，我希望资产模块各页面的顶部操作栏、行内操作和详情抽屉动作区遵循统一规则，  
以便在不同页面之间切换时具有一致的操作预期，减少认知成本和后续维护成本。

## 验收标准

- [x] 资产台账、业务单据、维修管理、盘点任务、分类管理、位置管理等主要页面的顶部操作栏遵循统一的按钮层级规则。
- [x] 行内操作统一为“最多 3 个高频动作 + 更多菜单”的结构，避免单行按钮无限膨胀。
- [x] 删除、驳回等不可逆操作统一使用危险样式，流程动作不再滥用危险样式。
- [x] 订单、维修、盘点等详情抽屉的动作区位置统一，避免同类页面出现不同的动作布局模式。
- [x] 不改变现有权限控制、业务状态流转和接口契约。

## 技术说明

- 建议优先提炼一套可复用的操作栏规则，而不是做成泛化的视觉重构。
- 列表页顶部主操作保留按钮，次级动作可使用 plain / warning plain / danger plain 的统一语义。
- 行内操作建议抽取共用配置，必要时收敛到“更多”菜单，避免 order、repair、inventory 页面继续膨胀。
- 详情抽屉建议统一动作区位置，优先与订单、维修、盘点保持一致。

## 依赖

- `asset/info`、`asset/order`、`asset/repair`、`asset/inventory`、`asset/category`、`asset/location`
- 现有资产按钮、抽屉和表格操作组件

## 实现结果

- 新增 `AssetRowActionBar` 作为资产模块统一的行内操作栏，默认展示 3 个直接动作，其余动作收进“更多”菜单。
- 分类、位置、台账、单据、维修、盘点页面已改为统一的文本操作按钮结构。
- 盘点详情抽屉的动作区已迁移到底部 footer，与订单、维修详情抽屉保持一致。
- 单据创建页的桥接上下文类型已补齐，避免页面回流时出现类型缺失。
