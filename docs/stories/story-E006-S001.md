# 资产新增页面化

**ID：** STORY-E006-S001
**史诗：** E006 页面化与模板化能力
**优先级：** Must Have
**故事点：** 8

## 用户故事

作为资产管理员，我希望在独立页面中新增资产，以便在不依赖大弹窗的情况下完成复杂录入、草稿保存和附件补录。

## 验收标准

- [ ] 可以从资产台账列表或新增入口进入独立新增页面，默认不再依赖大弹窗。
- [ ] 页面按基础信息、归属信息、价值信息、扩展字段和附件区域分区展示。
- [ ] 支持保存草稿、继续编辑和取消返回。
- [ ] 分类切换后可以刷新依赖字段，页面保留扩展字段接入点。
- [ ] 现有列表、详情、编辑、导出和权限口径不受影响。

## 技术说明

- 复用现有 `asset_info` CRUD 接口、分类树和位置选择器。
- 页面结构要优先保证清晰分区，避免继续把所有字段堆在一个弹窗里。
- 扩展字段区先预留统一接入口，待 `STORY-E006-S003` 接入后可直接渲染。
- 快捷新增入口如果保留，应作为页面的快捷跳转，而不是主交互。

## 依赖

- `asset_info` 现有 CRUD API
- `asset_category` 树查询
- `asset_location` 选择器
- `asset_attachment` 上传/抽屉能力
