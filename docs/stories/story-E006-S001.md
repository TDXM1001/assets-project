# 资产新增页面化
**ID:** STORY-E006-S001
**史诗:** E006 页面化与模板化能力
**优先级:** Must Have
**故事点:** 8

## 用户故事

作为资产管理员，我希望在独立页面中新增资产，以便在不依赖大弹窗的情况下完成复杂录入、草稿保存和附件补录。

## 验收标准

- [x] 可以从资产台账列表或新增入口进入独立新增页面，默认不再依赖大弹窗。
- [x] 页面按基础信息、归属信息、价值信息、扩展字段和附件区域分区展示。
- [x] 支持保存草稿、继续编辑和取消返回。
- [x] 分类切换后可以刷新依赖字段，页面保留扩展字段接入点。
- [x] 现有列表、详情、编辑、导出和权限口径不受影响。

## 技术说明

- 复用现有 `asset_info` CRUD 接口、分类树和位置选择器。
- 独立页面采用隐藏菜单路由接入，不新增前端静态路由模块。
- 扩展字段继续使用分类模板接入点，后续可直接承接 `STORY-E006-S003` 的模板能力。
- 草稿采用前端本地保存，不引入服务端草稿态，避免和现有校验逻辑冲突。
- 保存成功后返回 `assetId`，便于继续编辑和附件补录。

## 依赖

- `asset_info` 现有 CRUD API
- `asset_category` 树查询
- `asset_location` 选择器
- `asset_attachment` 上传/抽屉能力
