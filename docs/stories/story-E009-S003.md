# 类型化主档模型改造

**ID:** STORY-E009-S003  
**史诗:** E009 资产类型底座与差异化框架  
**优先级:** P0  
**故事点:** 5

## 用户故事

作为系统设计者，我希望资产主档支持统一核心字段加类型扩展字段的模型，以便不同类型资产都能挂在同一套底座上，同时保留各自差异。

## 验收标准

- [ ] 主档能够明确区分核心字段和类型扩展字段。
- [ ] 固定资产、不动产、低值易耗品都可以共用同一主档入口。
- [ ] 类型差异字段可以按模板保存和读取。
- [ ] 现有固定资产数据保持兼容，不会因为模型改造丢失数据。
- [ ] 保存、查询、编辑、导出等关键链路能正确处理类型差异。

## 技术说明

- 主档模型建议保持稳定，类型差异通过扩展字段或受控模板承载。
- 不建议把不动产、低值易耗品的全部差异直接堆进 `asset_info` 固定列。
- 需要兼顾历史数据兼容、报表稳定性和后续扩展能力。
