# 不动产空间结构管理

**ID:** STORY-E010-S003  
**史诗:** E010 不动产管理  
**优先级:** P1  
**故事点:** 5

## 用户故事

作为资产管理员，我希望能维护宗地、楼栋、楼层、房号等空间层级，以便描述不动产的真实空间关系，并让使用和出租都基于空间展开。

## 验收标准

- [ ] 支持空间层级树结构展示。
- [ ] 支持空间与不动产主档建立关联关系。
- [ ] 支持记录空间占用、空置和可用状态。
- [ ] 空间层级切换后能正确回显关联数据。
- [ ] 该能力可以支撑后续使用、出租和巡检场景。

## 技术说明

- 不动产对象不是单件，而是空间层级对象，因此空间树应成为核心能力。
- 空间结构建议复用树形组件和统一分类选择模式。
- 若后续要接更多空间类型，当前层级设计应保留扩展位。

## 前置依赖

- `STORY-E010-S001`
- 现有树形选择和树表展示能力

## 完成定义

- [ ] 代码完成并通过编译
- [ ] 空间树和主档关联测试通过
- [ ] 文档同步更新
- [ ] 代码审查通过
- [ ] 不同层级空间可正确回显和选择
