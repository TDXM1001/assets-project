# 调拨、归还与报废单据闭环

**ID：** STORY-E002-S002  
**史诗：** E002 单据主链路  
**优先级：** Must Have  
**故事点：** 5

## 用户故事

作为资产管理员，我希望完成调拨、归还与报废单据闭环，以便资产在不同业务状态之间切换时，台账和流水保持一致。

## 验收标准

- [x] 可以创建、提交、审批、驳回、完成和取消调拨、归还、报废单据。
- [x] 调拨完成后责任部门、责任人和位置会同步更新，并保留前后快照。
- [x] 归还完成后资产状态可以回到闲置、在用或维修相关状态。
- [x] 报废完成后资产进入已报废状态，不能继续流转。

## 前端核查

- 已补充归还单“归还后状态”选择（闲置/在用/维修中），用于填充明细 afterStatus，默认闲置。

## 技术说明

- 复用通用单据状态机，但要为不同单据类型保留独立规则。
- 调拨和报废的资产回写要保证事务一致性。
- 对异常分支要给出明确提示，避免卡死流程。
- 归还单在前端增加“归还后状态”入口，支持闲置、在用和维修中三种回写；`asset_status` 字典已补充 `REPAIRING`。
