# 类型化单据路由与页面壳

**ID:** STORY-E009-S004  
**史诗:** E009 资产类型底座与差异化框架  
**优先级:** P1  
**故事点:** 3

## 用户故事

作为资产管理员，我希望不同类型资产进入统一工作台壳后能自动加载对应单据内容，以便后续新增不动产和低值易耗品时不需要重复做页面壳。

## 验收标准

- [ ] 单据入口能按资产类型进行路由区分。
- [ ] 草稿和返回列表逻辑能按类型保留正确上下文。
- [ ] 页面壳统一，内容按类型差异化渲染。
- [ ] 不同类型进入工作台时，状态标签和说明文案能正确切换。
- [ ] 现有固定资产单据流程不受影响。

## 技术说明

- 单据页面建议维持统一壳层，类型差异下沉到配置层。
- 路由层应尽量保持稳定，只在类型入口和页面上下文中做区分。
- 后续不动产和低值易耗品单据都应该复用这套壳。
