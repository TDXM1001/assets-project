# 单据新增页面化

**ID：** STORY-E006-S002
**史诗：** E006 页面化与模板化能力
**优先级：** Must Have
**故事点：** 8

## 用户故事

作为资产管理员，我希望在独立页面中创建资产单据，以便把入库、领用、调拨、归还和报废统一到同一套页面化流程里。

## 验收标准

- [x] 可以从单据列表或新增入口进入独立单据页面，不再主要依赖弹窗。
- [x] 页面能根据单据类型切换头信息和明细区内容。
- [x] 支持草稿、提交、审批通过、驳回、取消和完成等动作。
- [x] 明细区支持资产选择和行级信息编辑。
- [x] 单据提交和完成后，仍然能正确写入事件流水并联动台账状态。

## 技术说明

- 复用现有 `asset_order` 头表 + 明细表模型。
- 页面结构建议统一成一套布局，再按 `order_type` 渲染差异内容。
- 草稿与提交动作应保持清晰分离，避免用户误提交不完整单据。
- 详情页、审批弹窗和完成状态展示应与新增页保持同一业务口径。

## 依赖

- `asset_order` 现有头表和明细表
- 单据提交 / 审批 / 完成接口
- 资产选择能力
- 事件流水写入能力
