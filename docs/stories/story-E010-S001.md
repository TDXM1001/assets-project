# 不动产主档建档

**ID:** STORY-E010-S001  
**史诗:** E010 不动产管理  
**优先级:** P1  
**故事点:** 5

## 用户故事

作为资产管理员，我希望能够为不动产建立专门主档，以便记录房产、场地、楼栋、房间等基础信息，并让后续权属、使用和维护都围绕同一条主档展开。

## 验收标准

- [ ] 可以新增、编辑、查询不动产主档。
- [ ] 主档至少包含不动产编号、名称、类型、面积、用途、位置和状态。
- [ ] 支持附件上传或关联证照文件。
- [ ] 不动产主档与现有固定资产主档能够并存，不互相干扰。
- [ ] 不动产主档页面能复用统一类型化壳层。

## 技术说明

- 不动产主档应优先复用通用资产底座，不建议独立造一套完全不同的实体模型。
- 主档字段应围绕空间和权属展开，而不是围绕设备型资产展开。
- 该故事应为后续权属、空间、使用和维修故事提供统一入口。

## 前置依赖

- `E009` 资产类型底座与差异化框架
- 现有 `asset_info` 主档能力

## 完成定义

- [ ] 代码完成并通过编译
- [ ] 主档新增和编辑测试通过
- [ ] 文档同步更新
- [ ] 代码审查通过
- [ ] 旧固定资产流程不受影响
