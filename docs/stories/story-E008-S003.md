# 单据类型驱动的页面结构统一
**ID:** STORY-E008-S003  
**史诗:** E008 单据工作台与流程连续性  
**优先级:** Should Have  
**故事点:** 3

## 用户故事

作为资产管理员，我希望不同单据类型能驱动页面结构差异，而不是靠散落的 if/else 来拼页面，以便后续继续扩展单据类型时仍然保持可维护性。

## 验收标准

- [ ] 不同单据类型可以映射到不同的头部文案、状态标签和默认说明。
- [ ] 页面结构差异可以通过配置或小型映射控制，而不是把逻辑堆在创建页组件里。
- [ ] 新增单据类型时，只需要补充配置，不需要重写工作台壳。
- [ ] 页面结构调整不会破坏现有单据类型的提交、审批和完成流程。

## 技术说明

- 建议把单据类型到页面结构的映射收敛到一个独立配置文件。
- 页面文案、标签和默认说明应该与单据类型绑定，但不与具体路由硬耦合。
- 如果后续单据类型继续增加，这一层应成为主要扩展点，而不是继续改 `create/index.vue` 的分支逻辑。
