# 单据工作台统一壳
**ID:** STORY-E008-S001  
**史诗:** E008 单据工作台与流程连续性  
**优先级:** Must Have  
**故事点:** 5

## 用户故事

作为资产管理员，我希望单据创建、详情和审批都沿用同一套工作台壳和动作规则，以便在系统内完成完整的单据流转，而不是面对几个风格割裂的独立页面。

## 验收标准

- [ ] 创建页、详情页和审批页共享统一的顶部状态区、主信息区、明细区和底部动作区。
- [ ] 页面能够明确展示当前单据类型、单据状态和来源上下文。
- [ ] 直接访问创建页时能够降级为通用新建单据，不依赖列表页状态。
- [ ] 仍然保留动态路由体系，不新增静态路由模块。
- [ ] 页面壳统一后，不影响现有权限控制、单据列表和事件流水。

## 技术说明

- 建议从 `asset/order/create/index.vue` 抽出工作台壳组件，避免创建页继续堆叠页面级逻辑。
- 单据工作台应与现有 `AppLayout` 保持一致，不能做成脱离系统壳的独立窗口。
- 顶部状态区、明细区和底部动作区应保持职责分离，避免一个文件同时处理路由、表单和审批逻辑。
