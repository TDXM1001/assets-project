## Sprint Goal

把资产模块里分散的顶部操作栏、行内操作和详情抽屉动作区统一成一套稳定规则，  
让不同页面之间的交互更一致，也为后续功能扩展保留可复用的操作模式。

## Planning Summary

- 盘点模块的主闭环已经足够完整，可以先不动。
- 当前最明显的交互不一致，集中在业务单据、维修管理和盘点任务的操作栏。
- 这轮 sprint 不做泛化视觉重构，只做可落地的操作语义统一。

## Existing Foundations

- `asset/info`、`asset/order`、`asset/repair`、`asset/inventory` 已经具备完整业务链路。
- `asset/category`、`asset/location` 的树表操作模式已经较稳定。
- 现有权限、状态流转和业务接口已经可用，不需要为统一操作栏重做后端契约。

## Out of Scope This Sprint

## Done Criteria

- 主要资产页面的顶部操作栏语义统一。
- 行内操作收敛到“高频动作 + 更多菜单”的结构。
- 订单、维修、盘点详情抽屉动作区位置统一。
- 现有权限、业务状态和接口契约不被破坏。
